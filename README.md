## 1. О проекте "Jobix"

**Jobix** — это IT-стартап, нацеленный на решение проблемы "информационной асимметрии" на рынке труда. Мы разрабатываем сервис, который помогает соискателям проводить "обратную проверку" работодателя еще до собеседования, анализируя вакансии на скрытые риски и "красные флаги".

Данный репозиторий содержит **интерактивную финансовую модель** этого проекта, выполненную в рамках курса "Финансовое моделирование" в МФТИ (ТехПред).

Цель модели — не просто спрогнозировать доход, а создать **динамический симулятор** для проверки бизнес-гипотез и анализа ключевых факторов риска.

## 2. Логика и Ключевые Предпосылки Модели

Эта модель — не "конь в вакууме". Она построена на ряде ключевых предпосылок, которые отражают нашу продуктовую и маркетинговую стратегию.

### Почему выручка считается только от **новых** пользователей?

Наша бизнес-модель основана на решении **краткосрочной, проектной** задачи — поиска работы.
*   **Жизненный цикл клиента:** Мы предполагаем, что активная фаза использования нашего продукта длится в среднем **3 месяца**. После успешного трудоустройства клиент "уходит в спячку" примерно на 2 года.
*   **Модель монетизации:** Мы не используем подписку, а продаем **единоразовые пакеты "алмазов" (анализов)**.
*   **Следствие:** Наша модель сфокусирована на **постоянном привлечении новых платящих клиентов**, а не на удержании старых. Поэтому выручка в каждом месяце напрямую зависит от количества **новых** пользователей, пришедших в этом месяце.

### Почему выручка имеет "рваный" рельеф?

График выручки отражает **сезонность рынка труда**.
*   **Пики (Январь-Февраль, Сентябрь-Октябрь):** Это периоды, когда люди активно ищут работу после праздников или летних отпусков. В эти месяцы мы прогнозируем **рост органического трафика на 150%**.
*   **Инвестиции в маркетинг (Август, Декабрь):** Чтобы усилить эффект от сезонных пиков, мы закладываем **целевые маркетинговые бюджеты** именно в месяцы, предшествующие им. Это приводит к резкому росту расходов, но и обеспечивает максимальный приток пользователей в следующем периоде.

### Почему NPV положителен, а IRR — низкий?

*   **NPV > 0 (~4.3 млн руб.)** показывает, что проект, даже с учетом высокой ставки дисконтирования в 25%, является **экономически целесообразным** и создает ценность.
*   **IRR ~ 6.8% годовых** — значительно ниже ставки дисконтирования. Это **не ошибка**, а **важный индикатор**. Он показывает, что проект **высокорискованный** и его прибыльность не является "взрывной" на старте. Успех сильно зависит от способности команды удерживать низкую стоимость привлечения и эффективно конвертировать пользователей, что и было подтверждено сценарным анализом.

## 3. Технологический стек


*   **Язык:** Python 3.11
*   **Библиотеки:**
    *   `streamlit` - для создания веб-интерфейса
    *   `pandas` - для обработки данных и расчетов
    *   `numpy` & `numpy-financial` - для финансовых вычислений (NPV, IRR)

## Как запустить локально

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/m2006342-lang/jobix-financial-model
    cd jobix-financial-model
    ```

2.  **Создайте и активируйте виртуальное окружение:**
    ```bash
    python -m venv venv
    .\venv\Scripts\activate
    ```

3.  **Установите зависимости:**
    ```bash
    pip install -r requirements.txt
    ```

4.  **Запустите приложение:**
    ```bash
    streamlit run dashboard.py
    ```
